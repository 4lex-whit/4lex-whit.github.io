Multilevel Feedback Queue

  1. How it works
  2. Advantages
  3. Disadvantages
  4. Operating Systems

How it works

  • Similar to Multilevel Queue Scheduling, but processes can move between queues
  • There are multiple queues, each with different time quantums/slices that are greater for lower priority queues
  • Processes start in the highest priority queue
  • If a process takes longer than the queue's time quantum/slice, it's moved to the lower priority queue
  • Processes in lower priority queues can only execute when all higher priority queues are empty
  • Processes executing in lower priority queues are interrupted by new processes in higher level queues
  • If a process waits too long in the lowest priority queue, it's moved to the highest priority queue and forced to execute
  • It's a preemptive algorithm - it will interrupt what's in the CPU

Advantages

  • It prevents starvation - when a process is constantly waiting for other processes to finish executing

Disadvantages

  • It produces more CPU Overheads - time spent with no useful work done
  • It's a very complex algorithm
